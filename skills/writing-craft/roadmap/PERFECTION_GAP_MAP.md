# Perfection Gap Map / 从“可用”到“用得好”

## Correction: no subjective-user-taste dependency
用户拒绝以个人主观喜好作为标准。正确路线改为：

1. 以公开/教材化 craft 知识补强为主；
2. 以本地已有经典/轻小说/VN/电影/漫画案例资料做客观机制拆解；
3. 用户只负责补“可被学习的资料”：书、教材、作品文本、剧本、案例，不需要给个人喜好判断；
4. 质量标准使用 craft rubrics、case-family patterns、失败诊断，而不是“用户喜欢/不喜欢”。

## Current honest status
当前还不是“完美”。它有路线、教材规则、模板、验证闸门、质量 rubrics 和 seed cases，但还缺深层案例库、风格-结构合流、更多专业教材维度。

## Gap 1: 案例库深度不足
本地已有很多 normalized notes，但尚未转成统一 case database。

Need:
- 每个作品/章节的 story engine、scene engine、dialogue engine、hook、turn、payoff 拆解；
- 横向 pattern family；
- anti-copy rule；
- 失败/成功条件。

## Gap 2: 教材维度缺口
已有：故事结构、冲突悬念、Save the Cat、Screenplay、Dialogue。

仍缺专业维度：
- prose style / sentence craft / concrete detail
- scene & sequel / MRU / stimulus-response
- character arc / moral psychology
- genre-specific design: mystery, romance, fantasy, YA/light novel, horror/thriller
- revision / self-editing / developmental editing
- worldbuilding with narrative pressure
- comedy / banter / voice
- emotional design / affective payoff

## Gap 3: 风格库未合流
已有物语、空境、月姬、Re:Zero 等 style guides，但还没强制映射到：信息释放、场景推进、对白策略、段落节奏、视角距离。

## Gap 4: 项目闭环不足
已有 relic-expedition 项目，但尚未系统化为：旧稿→诊断→修订→规则。

## Objective improvement route
1. 建 objective supply list：用户只补书/作品/剧本/案例，不补主观审美。
2. 建 local corpus inventory：明确手头已有资料能覆盖什么。
3. 建 repertoire deep-pass protocol：把已有 notes 全部转 case entries。
4. 建 craft expansion bibliography：缺哪些教材，用户可准备。
5. 每补一本/一组案例，就抽 concepts/action_rules/cases/patterns，然后验证。
