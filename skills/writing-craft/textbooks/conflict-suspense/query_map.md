# Query Map

- If `故事开端平静、人物无压力时`, query `麻烦缠身的人物`; apply gate: 第一页或第一场能指出具体麻烦及其对人物的风险。
- If `目标失败看起来无所谓时`, query `死亡赌注三分法`; apply gate: 主角失败会失去生命、安全、事业/身份，或完整自我中的一项。
- If `主线松散或缺少推进时`, query `LOCK基础`; apply gate: 四项各能用一句话说明；缺一项则先补基础再扩写场景。
- If `开头先铺背景、解释设定时`, query `以乱子开头`; apply gate: 开头场面中有行动中的人物和扰乱其日常的事件。
- If `主角可以轻易退出第二幕时`, query `无回关口`; apply gate: 读者能说明主角为什么不能回到原状。
- If `场景只是聊天、说明或气氛时`, query `场景目标障碍结果`; apply gate: 场景结束后局势变化；删掉该场会破坏因果链。
- If `中段疲软、桥段堆积时`, query `行动与反应节拍`; apply gate: 每个反应场景都连接总体目标与下一步行动。
- If `支线独立漂浮时`, query `子情节入侵主线`; apply gate: 至少两处能看到支线改变主角对主线的行动或代价。
- If `对话问什么答什么、只传信息时`, query `对话制造冲突`; apply gate: 每段关键对话后，关系、信息或权力位置发生偏转。
- If `读者没有继续翻页的问题时`, query `宏观悬念`; apply gate: 每章结尾至少留下一个与赌注相关的未解决问题。
- If `章节结尾疲软或重复同一种危机时`, query `悬崖挂壁变化`; apply gate: 结尾迫使读者关心后果，但不破坏全书最终满足感。
- If `场景中间缺少牵引时`, query `微观障碍与张力`; apply gate: 障碍虽小，却会让人物目标更难、时间更紧或秘密更危险。
