# Query Map

- If `主角不抓人或不能带动情节时`, query `值得写的主角三件套`; apply gate: 三项都能写成一句话，且 Want 与 Need 存在张力。
- If `故事只是事件串联时`, query `情节结构角色转变三位一体`; apply gate: 每个大事件都能说明它如何逼近或阻碍主角转变。
- If `长篇结构迷路或节奏失控时`, query `15动点路线图`; apply gate: 每个动点不是标签，而能对应具体场景/段落功能。
- If `触发事件只是气氛变化时`, query `催化剂撞出原轨道`; apply gate: 催化剂后主角若不行动，损失会扩大或局面不可逆。
- If `第二幕缺少新世界和方向时`, query `进入第二幕是错误方案`; apply gate: 第二幕世界与第一幕有明显差异，且主角策略带着缺陷。
- If `支线或关系线与主题无关时`, query `故事B承载内在旅程`; apply gate: 故事B删除后，主角内在转变会变薄或失去触发。
- If `中点只是普通大场面时`, query `中点改变压力级别`; apply gate: 中点后故事不能按原速度和原规则继续。
- If `中后段只增加外部麻烦时`, query `坏人逼近双线升级`; apply gate: 每次外部升级都暴露主角缺点或错误信念的新裂缝。
- If `低谷不够重或只是失败一场时`, query `一切尽失击碎旧我`; apply gate: 主角不能靠第一幕/第二幕的旧策略合理解决低谷。
- If `结尾只是打赢外部敌人时`, query `进入第三幕靠新选择`; apply gate: 最终行动同时解决外部目标并证明 Need 已被领会。
- If `类型承诺模糊时`, query `十种故事类型元素`; apply gate: 读者能清楚感到该类型的前提承诺，并在结尾获得对应满足。
- If `无法清楚推销故事时`, query `一句话简介模板`; apply gate: 一句话包含主角缺陷、第二幕动作、中点转折、失去一切风险和主题学习。
