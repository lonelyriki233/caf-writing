# Action Rules

## value-charge
Trigger: scene event is factual only

Rule: Track value change from positive to negative or reverse; story movement is value movement.

SOP gate: scene_card gate: identify value at start/end.

## beat-as-action-reaction
Trigger: beat list is vague

Rule: Break scenes into action/reaction beats where tactics change under resistance.

SOP gate: scene_card gate: beat turns required in key scenes.

## turning-point
Trigger: plot point is just event

Rule: A turning point must reverse expectation, reveal gap, or force new action.

SOP gate: volume_structure gate: plot point changes strategy/pressure.

## progressive-complication
Trigger: middle repeats pressure

Rule: Escalate complications by making each attempt harder, more personal, or more costly.

SOP gate: volume_structure gate: no flat repetition.

## gap-between-expectation-result
Trigger: events feel predictable

Rule: Design gaps between what character expects and what reality delivers.

SOP gate: scene_card gate: result should surprise character logically.

## crisis-choice
Trigger: climax is external only

Rule: Force a crisis choice between irreconcilable goods/bads before climax action.

SOP gate: volume_structure gate: final choice cost required.

## controlling-idea
Trigger: theme is slogan

Rule: Express theme as value plus cause proven by action outcome.

SOP gate: theme_action gate: theme tested by consequences.

## exposition-as-ammunition
Trigger: backstory dumps

Rule: Use exposition only when it arms current conflict or changes interpretation.

SOP gate: draft gate: info must have now-function.
