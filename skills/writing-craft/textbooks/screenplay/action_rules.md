# Syd Field Screenplay — action rules

Source:
- `/mnt/f/创作/奇迹/novel_lab/extracted/Screenplay_The_Foundations_of_Screenwriting_Syd_Field.txt`
- `/mnt/f/创作/奇迹/novel_lab/新增教材收束笔记_Screenplay_Dialogue_2026-04-22.md`

Purpose:
Convert Field's screenplay craft into novel_lab execution gates for chapter/sequence/scene design. This module is for planning and revision, not literary commentary.

## 0. Core conversion

Field's screenplay terms become novel-engineering terms:

- Subject -> `character + action`
- Dramatic need -> what the protagonist must do now, with stakes
- Act I / Setup -> establish protagonist, situation, relationships, imbalance
- Act II / Confrontation -> pursue need through escalating obstacles
- Act III / Resolution -> solve the core action line through consequence/choice
- Plot point -> event/decision/information that spins action into a new direction
- Scene -> local action unit with target, obstacle, result
- Sequence -> a block of scenes unified by one action idea
- Exposition -> minimum necessary information released through action pressure

## 1. Subject gate: no action, no story

Before drafting any chapter or arc, write:

```text
Subject:
This chapter/arc is about [character] in [situation/place] doing [observable action] in order to [dramatic need].
```

Hard failures:
- The sentence says only "introduce the guild", "show the relic system", "explain the continent", "establish atmosphere".
- The acting character is vague: "the world", "the kingdom", "the plot".
- The action is not observable: "learns lore", "understands history", "feels confused" without an external choice.

Pass condition:
- A reader can say who wants what, what the person does, and what changes if they fail.

## 2. Dramatic need gate

Every chapter needs one dominant dramatic need:

```text
Protagonist must: ______
Because if not: ______
Current obstacle: ______
Immediate next action: ______
```

Rules:
1. Need must be urgent enough to force action now.
2. Need must organize tasks, relationships, exposition, and reversals.
3. Need must not be replaced by curiosity alone. Curiosity can pull the reader, but the character still needs pressure.

Failure examples:
- The protagonist wanders through the guild to learn how it works.
- A mentor explains relic ethics while the protagonist passively listens.
- A mission exists, but nothing personal, relational, material, or moral is at stake.

## 3. Act structure as pressure phases

Use acts as function, not formula.

### Act I / Setup
Must establish:
- main character
- normal order or starting condition
- dramatic premise: what kind of problem this story is about
- dramatic situation: the circumstances around the action
- key relationships
- opening imbalance

Novel gate:
- First chapter / opening sequence must give action, sensation, and problem before deep background.
- The first ~3000 Chinese characters should include protagonist, current goal, real obstacle, and a question that pulls the next chapter.

### Act II / Confrontation
Must dramatize:
- pursuit of dramatic need
- repeated obstacles
- old strategy failure
- rising cost
- complications that cannot be solved by more explanation

Novel gate:
- Each sequence in the middle asks: what does the protagonist try, why does it partly fail, what new pressure does that create?

### Act III / Resolution
Must provide solution, not merely ending.

Novel gate:
- The resolution answers the core action line: succeed/fail/partial success/changed goal/price paid.
- A final scene may remain quiet, but the action line must be solved or deliberately transformed.

## 4. Plot point gate

A plot point is not a big event. It is an event, episode, decision, discovery, or reversal that hooks into the action and spins it into another direction.

For each planned plot point, fill:

```text
Before the plot point, protagonist is trying to: ______
The incident/decision/discovery is: ______
After it, protagonist must now: ______
What cannot return to normal: ______
Changed axis: target / risk / relationship / information / moral cost
```

Pass condition:
- After the plot point, the protagonist cannot continue the same plan under the same assumptions.

Hard failures:
- The plot point is only a spectacle.
- The reader learns something but the protagonist's action does not change.
- The event is reversible by ignoring it.
- The turn belongs entirely to an offstage antagonist and does not yet affect the main character.

Cadence rule for long fiction:
- Every 3-5 chapters should contain a real turn that changes at least one of: goal, risk, relationship, information, cost.

## 5. Scene unit gate

A scene is a local unit of action. It should move the story forward, reveal necessary character information, or both.

Before drafting, write one scene card per scene:

```text
Scene:
Place/time:
POV / focus character:
Target:
Obstacle:
Tactic:
Result:
Necessary information released:
New question / turn:
```

Pass condition:
- The result is a visible state change: gain, loss, refusal, exposure, promise, threat, cost, reversal, mistaken assumption, new lead.

Hard failures:
- Scene exists only to explain worldbuilding.
- Scene ends with the same action situation as it began.
- Characters ask and answer information cleanly without pressure.
- The necessary information field contains three or more unrelated lore points.

## 6. Sequence design gate

A sequence is a series of scenes connected by one single idea, with beginning, middle, and end.

Plan a sequence before listing all scenes:

```text
Sequence name, 1-5 words:
Single idea / context:
Mini dramatic need:
Beginning scene:
Middle escalation:
End turn:
Scenes included:
What this sequence changes:
```

Rules:
1. Name the sequence with a concrete action idea: "relic registration", "failed escort", "funeral return", "guild test", "false witness".
2. Every scene must serve the sequence idea.
3. The sequence must end with a turn, not just a stop.
4. A chapter can contain one sequence or multiple small sequences, but each must be coherent.

Hard failures:
- A chapter is just a list of events by location.
- The sequence name is abstract: "worldbuilding", "relationships", "mood".
- No mini-climax or turn closes the sequence.

## 7. Exposition/action gate

Exposition must be timed to action need.

Use this release test:

```text
What must the reader know now to understand the current action?
Who has a reason to reveal it under pressure?
What can remain delayed?
What physical action, procedure, conflict, mistake, taboo, or consequence can carry it?
```

Rules:
- Give the minimum information required for the current decision.
- Prefer procedure, conflict, and consequence over author explanation.
- Attach worldbuilding to a character's target and obstacle.
- If a concept is sacred, dangerous, bureaucratic, or taboo, reveal it through behavior: forms, gestures, corrections, silence, honorifics, penalties, witnesses.

Hard failures:
- A receptionist explains the full relic system in paragraphs.
- A mentor lectures history before the protagonist needs it.
- Dialogue answers exactly what the protagonist asks, with no tactic, withholding, or pressure.

## 8. Action and character balance

Action sequences must be character sequences.

Before writing an action-heavy passage, answer:

```text
What does the character want in this action?
What fear/flaw/old strategy shapes the tactic?
What choice does the action force?
What does the action reveal that dialogue could not?
What is the valley/rest beat after the peak?
```

Pass condition:
- If another character were substituted into the same action, the scene would change.

Failure:
- Bigger monsters, explosions, chases, or magic replace meaningful choice.
- Character emotion is told after the action instead of shaping the action.

## 9. Chapter workflow

Use this before prose:

```text
Chapter job:
Subject = character + action:
Chapter dramatic need:
Act / sequence position:
Plot point function, if any:
Opening imbalance:
Sequence A goal/context:
Scene 1 target / obstacle / result / new question:
Scene 2 target / obstacle / result / new question:
Exposition: must know now / must delay:
Closing turn:
Next chapter question:
```

Revision priority:
1. Dramatic need
2. Plot point / sequence turn
3. Scene target-obstacle-result
4. Exposition timing
5. Dialogue action/subtext if relevant
6. Sentence style

Do not polish prose before the action spine passes.

## 10. Quick diagnostic checklist

A chapter fails the Screenplay module if any of these are true:

- No subject sentence in `character + action` form.
- No dramatic need or stakes.
- Opening delays protagonist/action/problem for background.
- Scenes have no target-obstacle-result.
- A scene only explains lore.
- More than one major information packet is dumped into a scene.
- A sequence cannot be named by one concrete action idea.
- Plot point does not change subsequent action.
- Midsection obstacles do not force new tactics.
- Resolution stops the event but does not solve the action line.
- Action can be performed by any interchangeable character with no change.
